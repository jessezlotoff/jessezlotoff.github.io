Sudoku: How it Works

by: Jesse Zlotoff

This guide aims to provide basic information about how the Sudoku app works. All of the code can be found on Github. It was written in/for Python 2.7.

The part you see is in HTML, in the sudoku.html file. This defines the table with a standarized name for each cell and all of the other elements on the page. It also includes some simple in-file CSS to format the table and the buttons. The file references a Javascript file inside the <script> tag. This HTML file is hosted on my Github blog.

The interactive parts of the app are handled by Javascript, in the sudoku.js file. These functions are assigned to specific buttons in the HTML file. Here, the standrized names of the cells are used by the code to manipulate the table programmatically. This file is stored in the same place as the HTML file.

The most involved function is getSetupPuzzle(). This pulls a random puzzle and its solution as JSON from an API using an asynchronous event. It then stores the result in global variables and populates the HTML table.

The API is implemented in Flask, (info here) in the my_app.py file. The code runs custom Python functions to generate the puzzle and converts it to a JSON object. The rest of the code sets up a simple API, accepting only GET requests, to serve up the puzzles. This code is hosted through PythonAnywhere; the API can be accessed via a browser by navigating here.

The sudoku puzzles are generated by a custom Python class, in the sudoku.py file. It uses a Pandas DataFrame to hold the data for a puzzle and various methods to edit and display them. To create a new puzzle, it generates a completely filled random grid (using depth-first-search) and then randomly removes a select number of squares. It checks that the newly created puzzle can be solved using some combination of simple and complex logic, defined by custom functions. This file is also stored on PythonAnywhere so that the API can use it.
